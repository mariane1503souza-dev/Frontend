 # Frontend

O Frontend refere-se à parte de um website ou aplicação com a qual o usuário interage diretamente. É tudo o que o usuário vê e manipula no navegador, como layouts, cores, fontes, imagens, botões e formulários. O desenvolvimento Frontend concentra-se na experiência do usuário (UX) e na interface do usuário (UI), garantindo que a aplicação seja visualmente atraente, intuitiva e responsiva em diferentes dispositivos .

Em termos mais técnicos, o Frontend é o código que é executado no lado do cliente, ou seja, no navegador do usuário. Ele é responsável por apresentar os dados e permitir a interação, traduzindo as informações recebidas do servidor em uma interface gráfica compreensível para o usuário.


# Cliente x Servidor

O modelo Cliente-Servidor é a arquitetura fundamental que sustenta a maioria das aplicações web. Ele descreve a interação entre dois componentes principais:

•
Cliente: Geralmente, é o navegador web (como Chrome, Firefox, Safari) no dispositivo do usuário. O cliente envia requisições (pedidos) para o servidor, solicitando recursos como páginas web, imagens, vídeos ou dados. Ele também é responsável por renderizar e exibir a interface do usuário .

•
Servidor: É um computador remoto que armazena os arquivos do website, bancos de dados e a lógica de negócio da aplicação. Quando recebe uma requisição do cliente, o servidor processa essa requisição, busca as informações necessárias (muitas vezes em um banco de dados) e envia uma resposta de volta para o cliente. Essa resposta pode ser uma página HTML, dados JSON, ou qualquer outro recurso solicitado .

Essa arquitetura permite que a lógica de processamento e o armazenamento de dados sejam centralizados no servidor, enquanto o cliente se concentra na apresentação e interação, otimizando a distribuição de tarefas e o gerenciamento de recursos.


# Papel de HTML, CSS e JavaScript

HTML, CSS e JavaScript são as três linguagens pilares do desenvolvimento web Frontend, trabalhando em conjunto para criar a experiência que vemos nos navegadores:

•
HTML (HyperText Markup Language): É a linguagem de marcação padrão para criar páginas web. O HTML define a estrutura e o conteúdo de uma página, utilizando tags para organizar elementos como títulos, parágrafos, imagens, links e tabelas. Ele é o
esqueleto da página web .

•
CSS (Cascading Style Sheets): É a linguagem utilizada para descrever a apresentação de um documento escrito em HTML. O CSS controla o estilo visual da página, incluindo cores, fontes, espaçamento, layout, tamanhos e posicionamento dos elementos. Ele é a "pele e as roupas" que dão vida ao esqueleto HTML, tornando a página visualmente atraente .

•
JavaScript (JS): É uma linguagem de programação que adiciona interatividade e dinamismo às páginas web. Com JavaScript, é possível criar funcionalidades como validação de formulários, animações, manipulação de elementos HTML em tempo real, comunicação assíncrona com o servidor (AJAX) e muito mais. Ele permite que a página responda às ações do usuário sem a necessidade de recarregar a página inteira .


# W3C (World Wide Web Consortium)

O World Wide Web Consortium (W3C) é a principal organização internacional de padronização da World Wide Web. Fundado por Tim Berners-Lee (o inventor da Web), o W3C desenvolve e mantém padrões abertos para a web, como HTML, CSS, XML e muitas outras tecnologias. Seu objetivo é garantir a interoperabilidade e a evolução da web a longo prazo .

A importância do W3C reside em:

•
Padronização: Cria especificações técnicas que garantem que as tecnologias web funcionem de forma consistente em diferentes navegadores e dispositivos.

•
Acessibilidade: Promove a criação de uma web acessível para todos, incluindo pessoas com deficiência, através de diretrizes e tecnologias específicas.

•
Interoperabilidade: Assegura que diferentes sistemas e aplicações web possam se comunicar e funcionar juntos sem problemas.

•
Evolução: Lidera o desenvolvimento de novas tecnologias e aprimoramentos para a web, mantendo-a relevante e funcional para o futuro.

Ao seguir os padrões do W3C, desenvolvedores garantem que seus websites e aplicações sejam compatíveis, acessíveis e funcionem corretamente em uma ampla gama de ambientes .
